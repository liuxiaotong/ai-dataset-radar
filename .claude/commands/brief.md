# 快速情报简报

读取最新 Radar 报告，生成一份简洁的竞争情报简报。**不运行扫描**，纯读取分析，适合晨会快速浏览。

## 执行步骤

### 第一步：定位最新报告

在 `data/reports/` 下找到最新日期子目录中的 JSON 报告：

```bash
ls -t /Users/liukai/ai-dataset-radar/data/reports/*/intel_report_*.json 2>/dev/null | head -1
```

如果没有报告，提示用户先运行 `/scan`。

### 第二步：检查是否有 AI 分析报告

在同一日期目录下查找 `_insights.md`。

**如果存在**：直接读取，提取"关键发现"和"行动建议"两个章节。

**如果不存在**：进入第三步自行分析。

### 第三步：基于 JSON 数据生成简报

读取 JSON 报告，提取以下信息：

1. **数据集**：
   - 总数、类型分布
   - 按下载量 Top 5（含组织名、类别、下载量、发布日期）
   - 按 likes Top 5

2. **GitHub**：高相关仓库 Top 5（名称、stars、信号关键词）

3. **论文**：数据集相关论文列表（标题、日期、来源）

4. **X/Twitter**：活跃账号数、推文数

5. **博客**：文章数、来源分布

### 第四步：输出简报

用以下格式输出：

```
## AI 数据集情报简报 — {日期}

### 数据概况
| 维度 | 数量 |
|------|------|
| 数据集 | N |
| GitHub 仓库 | N |
| 论文 | N |
| 博客 | N |
| X 推文 | N |

### 关键发现
1. [最值得关注的事件，含具体数据集/组织名和日期]
2. ...
3. ...
4. ...
5. ...

### 行动建议
- [具体可执行的建议]
- ...

### 热门数据集 Top 5
| 数据集 | 组织 | 类别 | 下载量 | 发布日期 |
|--------|------|------|--------|----------|
| ... | ... | ... | ... | ... |
```

## 注意

- 此命令不触发网络请求，只读取本地文件
- 如果报告数据为空或异常，建议用户运行 `/scan --days 7` 更新数据
- 分析应具体、可执行，引用具体名称和数字，避免泛泛而谈
